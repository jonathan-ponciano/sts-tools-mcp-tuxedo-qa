MCP server · Playwright · Claude · Gemini

Let your AI agent own your QA loop.

tuxedo-qa is an MCP server that gives Claude, Gemini, and any other MCP-compatible assistant the tools to write, run, self-heal, and monitor Playwright tests for your web app — with a local dashboard, a credential vault, and a public status page, no CI pipeline required.

What it does

A single MCP server, connected to Claude, Gemini, or any MCP-compatible client, exposing 13 tools that cover the full lifecycle of a synthetic test suite.

✍️

Natural-language test authoring

Describe a flow, your AI agent writes the Playwright spec via create_test and saves it straight into the suite.

🩹

Self-healing runs

run_until_pass re-runs a failing test, lets your AI agent patch it between attempts, until it goes green or hits the retry limit.

⏱️

Scheduled monitoring

Attach a 1h / 6h / 24h schedule to any test to run it as a synthetic monitor, not just a one-off check.

🔐

Credential vault

Named credential sets (admin, user_comum, …) are stored once and injected into test runs — never pasted into prompts.

🛡️

Protection bypass headers

One-click presets for Vercel Protection Bypass, Cloudflare Access, and Basic Auth so tests can reach gated staging environments.

📟

Local dashboard

A zero-config web UI to browse tests, edit specs, trigger runs, and manage credentials without touching the CLI.

📣

Discord alerts

Configure a webhook once and every run posts pass/fail results straight to your team's channel.

📊

Public status page

Turn any subset of tests into a shareable uptime page with a 60-run history bar, for stakeholders who don't need MCP access.

🧑‍🤝‍🧑

Human-in-the-loop

Tests can pause mid-run and wait for a human-supplied value (2FA codes, OTPs) via a simple file-based signal.

Use cases

Built for teams that want real QA coverage without standing up a CI/CD testing pipeline.

1

Post-deploy smoke tests, driven from chat

Ask Claude or Gemini to run the suite right after a deploy — pause_tests beforehand to avoid false alerts, then run_tests once you're live.

2

Continuous synthetic monitoring

Schedule critical flows (login, checkout, lead capture) to run every hour and surface failures via Discord before customers notice.

3

Testing behind auth walls

Staging environments protected by Vercel/Cloudflare Access or Basic Auth are reachable via the built-in protection header presets.

4

Flows that need a human for one step

SMS/email 2FA steps pause the test and wait for a value you supply through your AI agent, then resume automatically.

5

A status page for non-technical stakeholders

Expose a curated set of tests as a public uptime page — no dashboard login, no MCP client required to view it.

MCP tools reference

Everything Claude, Gemini, or any other MCP client can call once the server is connected.

ToolWhat it does
create_testCreate a new Playwright test file from generated test code.
list_testsList all test files with their last known status; filter by status or limit.
read_testRead the contents of a test file.
update_testUpdate a test's script, display name, description, schedule, or enabled state.
delete_testPermanently delete a test and its history. Prefer disabling via update_test.
run_testsRun all tests or a specific file; respects pause state; notifies Discord if configured.
run_until_passRun a test repeatedly, applying fixes between attempts, until it passes or hits the retry cap.
get_statusOverall suite status, or per-test failure detail and fix suggestions.
pause_testsPause the whole suite for up to 60 minutes — auto-resumes.
set_webhookConfigure the Discord webhook URL used for run notifications.
create_credentialCreate or update a named credential set (key-value fields).
list_credentialsList saved credential sets with masked values.
delete_credentialDelete a named credential set.

Quickstart

Runs entirely on your machine — a local MCP server plus an optional dashboard.

Clone and install

git clone https://github.com/jonathan-ponciano/sts-tools-mcp-tuxedo-qa.git
cd sts-tools-mcp-tuxedo-qa
npm install
npm run build

Register it as an MCP server

Works with any MCP-compatible client. For example, Claude Code:

claude mcp add tuxedoqa -- node "$(pwd)/dist/index.js"

Or Gemini CLI:

gemini mcp add tuxedoqa -- node "$(pwd)/dist/index.js"

(Optional) start the local dashboard

npm run dashboard
# → http://localhost:3131

Ask your AI agent to create your first test

"Create a Playwright test that logs into staging and checks the dashboard loads" — Claude or Gemini calls create_test, saves the spec, and can run it immediately with run_tests.
